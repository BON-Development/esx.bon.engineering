Understanding the Schema

An Ekahau project file consists of many JSON files that link together using different IDs. Understanding the mapping of these JSON files is crucial when developing a script to interrogate these JSON files.

As an example (and using the graphic above), to discover the technologies used for a specific measured access point you need to know the following information:

  • Access Point Name
  • Access Point ID
  • Measured Radio ID

We find that most of the time, you begin within the accessPoint.json file. However, you are not limited to starting at this file. There are also JSON files that are not connected with accessPoints.json (project.json for example)

The best way to better understand the schema is to explore it. Head over to esx.bon.engineering and have a look around, alternatively take a look at the use cases shown on this website

Example Use Cases

Ekahau Cloud "User" Unifier

Many of our customers own Ekahau subscriptions and utilise the Ekahau Cloud. Ekahau Cloud will read the project file and display information such as who performed the survey or added a note.

At BON, we wanted all of our project files to show a uniform response to this query, and have developed a small script that will rename these fields to "BON Engineering" (Whereas it would normally display the name of the person the software is licensed to). This is seen in the screenshot above.

Ekahau Remote RF Checker

There are many caveats with this initiative, as you will identify yourself throughout the explanation.

Using a predictive design, update the transmit power and channel allocation of each access point. This is done by connecting to the wireless controller and retrieving the information weekly and updating the Ekahau project file.

This helps operation teams to monitor the status of the RF in a particular site based on real-world values

Some caveats to get you started:

  • Design must be validated against a passive survey to demonstrate and ensure accuracy
  • Initiative assumes the physical attributes of the site remain the same, with no major changes within the facility
  • Should not be used as a replacement to a passive survey (does not account for neighbouring access points for example)
