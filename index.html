What you're looking at

An .esx file is a graph, not a document.

Rename it to .zip and you'll find a folder of JSON files. None of them mean much alone — the value is in how they reference each other by ID. Learn the wiring once and every automation gets easier.

01

Many small files

Access points, radios, floor plans, notes, surveys — each concern lives in its own JSON document like accessPoints.json.

02

Linked by IDs

Files point at each other through id / ...Id fields. A measured radio knows its access point; an AP knows its floor plan.

03

Traversable

Follow the relationships and you can answer real questions — which technologies a measured AP supports, what floor a note sits on, and more.
